Technology Stack

Infrastructure Built
to Last.

No proprietary lock-in. No fragile integrations. The ADI™ stack uses open standards, public blockchains, and edge infrastructure that will still be running in 20 years. Everything is verifiable. Everything is replaceable. Nothing is magic.

The ADI Pipeline

RAW → FILE in Six Deterministic Stages

The canonical pipeline that transforms raw business data into deployed AI signals. Each stage is deterministic. No ambiguity. No AI in the delivery path.

RAW

Raw Business Data

Unstructured business information — website copy, service descriptions, team bios, certifications, and client data. The starting material.

FACT

Fact Extraction

Raw data is parsed into discrete facts. Each fact is categorised, validated, and tagged with source and confidence level. Ambiguous facts are flagged for human review (HITL).

ENTITY

Entity Construction

Facts are assembled into structured entities — business, services, team, location, certifications. Each entity has a canonical identifier and relationship map.

RELATIONSHIP

Relationship Mapping

Entities are connected into a knowledge graph. Service → sector → buyer intent. Business → founder → credentials. Location → market → competitors.

SIGNAL

Signal Generation

The entity graph is compiled into 167 atomic AI signals across three categories: Identity & Entity (48), Trust & Proof (61), Intent & Governance (58).

FILE

File Deployment

Signals are serialised into deployment files — ai.json, entities.json, intents.json, llms.txt, schema.jsonld — and pushed to Cloudflare Workers KV at the edge.

The Stack

Every Technology We Use and Why

No proprietary tools. Every technology in this stack is open, public, and independently verifiable.

Content Layer

Source of Truth

GitHub

All content files live in GitHub. Version controlled. Public. Deployable via git push. The canonical SSOT for all human-facing content.

github.com/aipath512
Hosting Layer

Static Site Hosting

Cloudflare Pages

GitHub → Cloudflare Pages auto-deploy on every push to main. Zero server management. Global CDN. Free tier covers all static deployments.

pages.cloudflare.com
Signal Layer

Edge Key-Value Store

Cloudflare Workers KV

167 AI signals stored as tenant-first key-value pairs. Served via Cloudflare Workers at the edge. Sub-millisecond latency globally. Zero origin server.

GET /v1/{tenant}/{file}
Execution Layer

Edge Computing

Cloudflare Workers

Serverless JavaScript execution at the edge. Handles signal routing, tenant isolation, and request transformation. Zero LLM in the execution path.

workers.cloudflare.com
Proof Layer

Cryptographic Hashing

SHA-256

Every core document is fingerprinted using SHA-256. The industry standard for cryptographic content verification. Deterministic. Collision-resistant. Universally supported.

sha256sum document.txt
Proof Layer

Timestamp Anchoring

OpenTimestamps

SHA-256 hashes are anchored to the Bitcoin blockchain via OpenTimestamps. Creates immutable, publicly verifiable proof of when a document existed. Open source. No trusted third party required.

opentimestamps.org
Proof Layer

Immutable Ledger

Bitcoin Blockchain

The Bitcoin blockchain is the anchor for all OpenTimestamps proofs. The most secure, most decentralised, longest-running public ledger in existence. Blocks 944442–952597.

bitcoin.org
Signal Layer

Structured Data Standard

Schema.org

The universal vocabulary for structured data. Used by all major AI systems and search engines to understand entity types, service categories, and relationship graphs.

schema.org
Signal Layer

Knowledge Graph Format

JSON-LD

JSON for Linked Data — the canonical format for entity graphs. Embedded in HTML or served as standalone files. The format that connects business entities to the global knowledge graph.

json-ld.org

Architecture Principles

Why We Built It This Way

Every architectural decision was made for a reason. Here are the six core principles.

🔀

Fully Decoupled

Site=CONTENT (GitHub). Edge=SIGNALS (Cloudflare KV). These never touch. Updating signals does not require touching the website. Updating the website does not affect signals.

⚡

Zero Origin Dependency

All signals are served from the Cloudflare edge via GET /v1/{'{'}tenant{'}'}/{'{'}file{'}'}. No origin server involved. No downtime. No CMS dependency. Sub-millisecond globally.

🏗️

Tenant-First at Scale

Every tenant key is prefixed: tenant:signal-file. Zero cross-tenant leakage. Onboarding is two steps: git add + CNAME. Designed to scale to 12M EU businesses.

🤖

Zero LLM in Delivery

The delivery edge is 100% deterministic. No AI processing in the serving path. LLMs are only permitted in the Signal Processing Layer during signal generation — never in delivery.

🔐

Cryptographic Proof at Every Layer

SHA-256 at the document layer. OpenTimestamps at the proof layer. Bitcoin at the anchor layer. Every claim is traceable to a mathematically verifiable origin.

🇪🇺

EU-First by Design

GDPR compliance is structural. EU AI Act readiness is built in. All data processing follows EU law. Cloudflare edge nodes in EU serve EU tenants. We support readiness. We never certify.
