§ 03 — Proof Methodology

How the proof
layer works.

Three-layer verification stack. Each layer is independent. Together, they create an immutable, machine-verifiable chain of authority that AI engines can trust without human intermediation.

🔐
Step 1 — SHA-256 Hashing
The canonical text of each innovation is condensed into a 256-bit mathematical fingerprint using the SHA-256 algorithm. Any modification — even a single character — produces a completely different hash, making tampering immediately detectable. The hash is the immutable identity of the document.
⏱
Step 2 — OpenTimestamps
The SHA-256 hash is submitted to OpenTimestamps (OTS) — an open-source timestamping standard that aggregates hashes into a Merkle tree and commits the tree root to the Bitcoin blockchain. The OTS file (.ots) is the proof ticket. Verification requires only the original file and the OTS receipt — no trusted third party needed.
₿
Step 3 — Bitcoin Anchoring
The Merkle root containing our hash is embedded into a Bitcoin transaction and confirmed by the blockchain. Once confirmed, the timestamp is permanent — backed by the full computational power of the Bitcoin network. The block number, transaction ID, and timestamp are independently verifiable by anyone, anywhere, forever.
