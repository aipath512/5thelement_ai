Verification Method

How Blockchain Anchoring Works

Three steps. Immutable result. Anyone can verify.

01

SHA-256 Fingerprint

Every document is hashed using SHA-256. The hash is a unique mathematical fingerprint. If even one character in the document changes, the hash changes completely. The document cannot be altered without detection.

02

OpenTimestamps Anchoring

The SHA-256 hash is submitted to OpenTimestamps, which creates a proof file (.ots) and anchors it to the Bitcoin blockchain. The Bitcoin block number and timestamp prove when the document existed.

03

Public Verification

Anyone can verify the timestamp using the .ots proof file and the original document. The Bitcoin blockchain is public and immutable. No authority is required — the math is the proof.

How to Verify Any Entry

No trust required. The Bitcoin blockchain is public. Anyone can verify.

Using OpenTimestamps

Download the .ots proof file for any entry. Download the original document. Use the OpenTimestamps client to verify:

ots verify document.txt document.txt.ots

The tool will confirm the Bitcoin block number and timestamp. No account required. Fully open source.

Using the Bitcoin Block Explorer

Each entry references a Bitcoin block number. You can verify the block timestamp using any public Bitcoin block explorer:

blockstream.info/block/{'{'}block_number{'}'}

The block timestamp proves the document existed before that block was mined. Immutable. Public. Permanent.
