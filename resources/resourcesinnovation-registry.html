AI Signal Files

Priority Discovery Signals

The eight core AI discovery files deployed on 5thelement.ai. Each serves a specific function in the AI knowledge graph.

AI Entity Definition

/ai.json

Primary AI identity file. Defines the entity, category, services, and canonical properties for AI systems crawling the site.

Entity Graph

/entities.json + /entity-graph.public.jsonld

Structured entity relationships in JSON-LD format. Enables AI systems to map the full business graph — founder, company, services, products, and properties.

Intent Router

/intents.json

Global intent router. Maps buyer queries to the correct module and response. Two-layer system — global router + module-level contracts.

LLM Instructions

/llms.txt

Canonical instructions for large language models accessing this domain. Defines what AI can read, reference, and cite — and what requires verification.

Crawler Permissions

/robots.txt

Standard crawler permissions combined with AI-specific directives. Allow-lane matrix for different AI systems and crawlers.

Sitemap

/sitemap.xml

Full site structure with priority weighting. Signals to both search engines and AI crawlers which pages carry the highest authority and freshness.

AI Proof File

/ai-proof.json + /ai-proof.json.ots

Cryptographic proof file. SHA-256 hash anchored to Bitcoin via OpenTimestamps. OTS verification file publicly accessible. Schema version 1.1 locked.

Allow Lane Matrix

/allow-lane-matrix.json

Defines which AI systems have access to which signal tiers. Governs the permission layer between public signals and tenant-specific edge delivery.
